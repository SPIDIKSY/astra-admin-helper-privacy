Privacy Policy

Astra Admin Helper Pro  ·  Extension ID: ofdiekkomjacbkaeoommjffdnglndhkg  ·  Last updated: May 19, 2025

1. Overview

Astra Admin Helper Pro ("the Extension") is a browser extension designed exclusively for internal use by authorized moderators and administrators of the Astra Manga platform (astramanga.org). Its purpose is to automate repetitive moderation tasks such as auto-filling manga metadata fields, parsing public data from manga aggregator websites, and enforcing in-house editorial rules.

✔ The Extension does not collect, transmit, sell, or share any personal information about end users. All data it handles is either public manga catalog data or locally stored operational settings.

2. Data Collected & How It Is Used

2.1 Data stored locally on your device Local only

The Extension stores the following data exclusively in chrome.storage.local (and localStorage as a fallback). This data never leaves your device and is not transmitted to any server.

Key What it contains Purpose Retention
am_ext__panel_state Panel width, height, scale, minimized flag Restore UI panel position & size across page loads Until user resets or uninstalls
am_ext__feature_flags Boolean toggles (e.g., useProxy, useMangadex, skipLinkChecks) Persist user-configured workflow settings Until user resets or uninstalls
am_ext__panel_settings Panel dimension preferences Persist user-configured panel dimensions Until user resets or uninstalls
am_ext__lib_mirror_urls_by_title_v1 Per-title list of mirror source URLs (user-entered) Remember which source mirrors the user linked to a specific manga title Until user clears or uninstalls
am_ext__am_spididy_dispatcher_v1 An extension authentication key (ext_key) Authenticate requests to the internal Spididy proxy API used by the team Until user signs out or uninstalls

None of these values contain personally identifiable information (PII) such as names, email addresses, passwords, or financial data. The authentication key is an opaque machine-generated token associated with the extension installation, not with a natural person.

2.2 Network requests made by the Extension Read-only / Public data

To populate manga metadata fields, the Extension makes outbound HTTP GET requests to the following categories of third-party services. These are read-only requests to publicly accessible catalog APIs and web pages:

  • Manga aggregators — MangaLib, HentaiLib, SlashLib, ShLib (api.cdnlibs.org, hapi.hentaicdn.org)
  • International databases — MangaDex (api.mangadex.org), MangaUpdates (mangaupdates.com)
  • Anime/manga metadata — Shikimori (shikimori.one / shikimori.me), MyAnimeList (myanimelist.net), Kitsu (kitsu.io), Anime-Planet (anime-planet.com)
  • Russian aggregators — Remanga (remanga.org), ReadManga, MintManga, NewManga, Senkuro, Inkstory, MangaBuff
  • Astra Manga internal API — internal.astramanga.org, api.astramanga.org
  • Spididy proxy — impidoras-spididy.waw0.amvera.tech (internal team proxy, used only for requests to MangaDex and MangaUpdates)

The Extension does not send user browsing history, form data, or any personal information to these endpoints. Requests contain only the URL being fetched and standard HTTP headers (Accept, User-Agent). The authentication key described in §2.1 is sent only to Astra-owned endpoints (astramanga.org and spididy.waw0.amvera.tech).

3. Why Each Permission Is Required

Permission Reason
storage Persist panel state, feature flags, and source URLs in chrome.storage.local
tabs Detect existing MangaLib/HentaiLib browser tabs to execute authenticated fetch requests for manga data that requires a logged-in session (read-only; tab content is not recorded)
scripting Inject a fetch function into an existing manga library tab to retrieve API responses that require session cookies (the result is returned to the admin page; no user data is retained or transmitted elsewhere)
Host permissions (astramanga.org, mangadex.org, etc.) Required by Manifest V3 to make cross-origin requests from the background service worker to the external catalog APIs listed in §2.2

4. Data Sharing & Third Parties

The Extension does not share, sell, license, or otherwise disclose any user data to third parties. The only outbound traffic the Extension generates consists of read-only requests to public manga APIs, as described in §2.2.

The Extension does not include any analytics SDK, advertising network, crash-reporting service, or any other third-party tracking library.

5. Data Security

All data processed by the Extension remains on the user's local machine inside the browser's sandboxed storage. The Extension communicates with Astra-owned endpoints over HTTPS. No data is stored on any remote server operated by the Extension author.

The authentication key is stored only in chrome.storage.local, which is accessible only to this Extension and cannot be read by web pages or other extensions.

6. Children's Privacy

The Extension is intended solely for use by adult professional staff (content moderators and administrators) of the Astra Manga platform. It is not directed at, and does not knowingly collect information from, children under the age of 13.

7. Changes to This Privacy Policy

If the Extension's data practices change materially, this page will be updated and the "Last updated" date at the top will reflect the revision date. Continued use of the Extension after any update constitutes acceptance of the revised policy.

8. Contact

Questions or concerns regarding this privacy policy may be directed to the developer via email: vladvasilebsk@gmail.com.

1. Общие сведения

Astra Admin Helper Pro («Расширение») — браузерное расширение, предназначенное исключительно для внутреннего использования авторизованными модераторами и администраторами платформы Astra Manga (astramanga.org). Цель расширения — автоматизация рутинных задач модерации: автозаполнение полей метаданных манги, парсинг публичных данных с сайтов-агрегаторов, а также применение внутренних редакционных правил.

✔ Расширение не собирает, не передаёт, не продаёт и не раскрывает никакие персональные данные пользователей. Все данные, с которыми оно работает, — это либо публичная информация из каталогов манги, либо локальные операционные настройки.

2. Собираемые данные и их использование

2.1 Данные, хранящиеся локально на устройстве Только локально

Расширение хранит перечисленные ниже данные исключительно в chrome.storage.local (с резервным вариантом через localStorage). Эти данные никогда не покидают ваше устройство и не передаются ни на какие серверы.

Ключ Содержимое Назначение Срок хранения
am_ext__panel_state Ширина, высота, масштаб и состояние (свёрнута/развёрнута) панели Восстановление положения и размера панели при перезагрузке страницы До сброса настроек или удаления расширения

am_ext__panel_settings Предпочтения размеров панели Сохранение пользовательских настроек размеров панели До сброса настроек или удаления расширения
am_ext__lib_mirror_urls_by_title_v1 Список URL-зеркал источников для каждого тайтла (вводятся пользователем) Запоминание, какие зеркала пользователь привязал к конкретному тайтлу манги До очистки или удаления расширения
am_ext__am_spididy_dispatcher_v1 Ключ аутентификации расширения (ext_key) Аутентификация запросов к внутреннему прокси-API Spididy, используемому командой До выхода из системы или удаления расширения

Ни одно из этих значений не содержит персональных данных (имён, адресов электронной почты, паролей, финансовой информации). Ключ аутентификации — это непрозрачный машинно-генерируемый токен, связанный с установкой расширения, а не с конкретным человеком.

2.2 Сетевые запросы расширения Только чтение / публичные данные

Для заполнения полей метаданных манги расширение выполняет исходящие HTTP GET-запросы к следующим категориям сторонних сервисов. Это запросы только на чтение к публично доступным API каталогов и веб-страницам:

  • Агрегаторы манги — MangaLib, HentaiLib, SlashLib, ShLib (api.cdnlibs.org, hapi.hentaicdn.org)
  • Международные базы данных — MangaDex (api.mangadex.org), MangaUpdates (mangaupdates.com)
  • Метаданные аниме/манги — Shikimori (shikimori.one / shikimori.me), MyAnimeList (myanimelist.net), Kitsu (kitsu.io), Anime-Planet (anime-planet.com)
  • Российские агрегаторы — Remanga (remanga.org), ReadManga, MintManga, NewManga, Senkuro, Inkstory, MangaBuff
  • Внутренний API Astra Manga — internal.astramanga.org, api.astramanga.org
  • Прокси Spididy — impidoras-spididy.waw0.amvera.tech (внутренний командный прокси, используется только для запросов к MangaDex и MangaUpdates)

Расширение не передаёт историю посещений, данные форм или какую-либо персональную информацию этим сервисам. Запросы содержат только запрашиваемый URL и стандартные HTTP-заголовки (Accept, User-Agent). Ключ аутентификации, описанный в п. 2.1, передаётся только на серверы Astra (astramanga.org и spididy.waw0.amvera.tech).

3. Зачем нужны каждое из разрешений

Разрешение Причина
storage Сохранение состояния панели, флагов и URL источников в chrome.storage.local
